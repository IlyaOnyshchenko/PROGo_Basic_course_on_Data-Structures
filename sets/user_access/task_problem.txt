Авторизация
В операционных системах семейства Unix пути к файлам имеют следующий вид: /folder/folder2/folder3/filename.ext. Они начинаются с символа / (слэш), затем идут имена вложенных папок, разделённые этим же символом. Например, в нашем примере папка folder3 вложена в папку folder2, которая, в свою очередь, вложена в папку folder. В конце, после последнего слэша, идёт имя файла и расширение (filename.ext), разделённые точкой.
Вам поручено написать систему распределения доступа к файлам. Она должна не пускать пользователя к тем файлам, прочитать которые у него не хватает прав.
Права пользователю можно выдавать как на отдельный файл, так и на целую папку. Например, можно разрешить читать только файл /folder/folder2/folder3/filename.ext, а можно разрешить читать всю папку /folder/folder2 и тогда пользователь сможет автоматически прочитать любой файл в этой папке, в любой её подпапке и так далее.
Вашей программе выдан список разрешений и запросов. Для каждого запроса выведите, имеет ли право пользователь читать этот файл. Пользователь имеет право читать файл, если ему явно разрешили читать именно этот файл, или если ему разрешили читать любую из папок, в которых находится этот файл. В примере выше пользователь может читать файл filename.ext, если ему разрешили читать его или если ему разрешили читать одну из папок /folder, /folder/folder2, /folder/folder2/folder3.
 
Входные данные
В первой строчке дано целое число N (1≤N≤1000) — количество выданных разрешений.
В следующих N строчках записаны пути, которые пользователю разрешено читать. Путь может содержать только латинские буквы (заглавные и строчные), символ подчёркивания (_), точку и слэш. Гарантируется, что подпапка и файл в одной папке не могут иметь одинаковое название.
В следующей строчке дано целое число M (1≤M≤1000) — количество запросов.
В следующих M строчках записаны пути, для которых запрашивается доступ пользователя.
Выходные данные
Для каждого запроса выведите YES, если пользователь сможет прочитать этот ресурс (то есть ему был выдан доступ к файлу или к одной из его родительских папок), и NO, если не сможет.

Sample Input 1:

1
/folder/folder2
4
/folder
/folder/folder2
/folder/folder2/folder3
/folder/folder2/folder3/filename.ext

Sample Output 1:

NO
YES
YES
YES

Sample Input 2:

1
/folder/folder2/filename.ext
4
/folder
/folder/folder2
/folder/folder2/folder3
/folder/folder2/folder3/filename.ext

Sample Output 2:

NO
NO
NO
NO
